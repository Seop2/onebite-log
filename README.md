# CHZZK Community

치지직 스트리머와 관련된 게시글을 작성하고, 이미지/영상 업로드, 댓글, 좋아요, 프로필 관리 기능을 제공하는 커뮤니티 서비스입니다.

단순 CRUD 구현을 넘어서, 인증, 스토리지 업로드, 권한 제어(RLS), 무한 스크롤, 캐시 관리까지 포함한 실서비스형 프론트엔드 프로젝트를 목표로 개발했습니다.

## 제작 배경

기존에는 UI 유지보수 중심의 경험이 많았기 때문에,  
직접 서비스 단위의 기능을 설계하고 구현하는 경험이 필요하다고 느꼈습니다.

그래서 단순한 화면 구현이 아니라, 실제 커뮤니티 서비스에서 자주 사용되는 기능인 인증, 게시글 작성, 미디어 업로드, 댓글, 좋아요, 프로필 관리까지 포함한 프로젝트를 만들었습니다.

특히 다음과 같은 경험을 쌓는 것을 목표로 했습니다.

- 사용자 인증과 세션 관리
- 이미지/영상 업로드 및 스토리지 연동
- 게시글/댓글/좋아요 데이터 모델링
- React Query를 활용한 서버 상태 관리
- Supabase RLS를 활용한 권한 제어

## 기술 선택 이유

### React

컴포넌트 단위로 UI를 분리하고 재사용 가능한 구조를 만들기 위해 사용했습니다.

### TanStack Query

서버 상태와 클라이언트 상태를 분리하고, 캐싱 / 무한 스크롤 / 비동기 로딩 상태를 효율적으로 관리하기 위해 사용했습니다.

### Zustand

모달 상태, 세션 관련 UI 상태처럼 전역에서 가볍게 관리해야 하는 상태를 처리하기 위해 사용했습니다.

### Supabase

인증, 데이터베이스, 스토리지를 빠르게 구축할 수 있어 MVP 개발에 적합하다고 판단했습니다.  
또한 RLS를 통해 사용자 권한 제어를 직접 구현해볼 수 있다는 점도 선택 이유였습니다.

## 폴더 구조

```bash
src
├─ api
├─ components
├─ hooks
│  ├─ queries
│  └─ mutations
├─ pages
├─ store
├─ lib
└─ types

### 폴더별 역할

> store: 전역 ui 상태관리 (zustand)

> api : 외부 비동기 함수, 서버 통신

> provider : 세션, 모달

> pages : 페이지별 라우터

> lib : 자주쓰는 상수값, 공통 유틸 함수

> hook > mutation : 서버에서 불러오는 데이터 수정 및 삭제
> hook > queries : 조회
```
